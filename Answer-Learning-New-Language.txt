Part 2: Four-Step Prompting Strategy

Topic Chosen: Java Streams

Step 1: Conceptual Understanding

Key Learnings:

1. Java Streams process collections declaratively.

2. Streams reduce boilerplate code compared to loops.

3. Streams focus on data transformations rather than iteration.

Step 2: Step-by-Step Breakdown

Key Learnings:

1. Streams use filter(), map(), and collect() operations.

2. Streams can be chained together.

3. Streams improve readability when processing collections.

Step 3: Guided Implementation

Example:

```java
List<String> names = Arrays.asList("John", "Sarah", "Mike");

List<String> filteredNames = names.stream()
        .filter(name -> name.startsWith("M"))
        .collect(Collectors.toList());
```

Step 4: Understanding Verification

Key Learnings:

1. Stream operations should remain simple.

2. Complex business logic should be extracted into methods.

3. Readability is more important than using streams everywhere.

Part 3: Advanced Prompting Techniques

Technique 1: Using Context Effectively

Key Learnings:

1. Python list comprehensions are conceptually similar to Java Streams.

2. Java Streams provide stronger typing.

3. Stream pipelines encourage functional programming concepts.

Technique 2: Promoting Deep Understanding

Key Learnings:

1. Streams can improve readability but are not always faster.

2. Parallel Streams can improve performance for large datasets.

3. Understanding performance trade-offs is important.
