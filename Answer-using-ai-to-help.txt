Part 2: Improved Unit Tests

Test 1: Priority Weight Calculation

```java
@Test
public void shouldGiveHigherScoreToUrgentTaskThanLowPriorityTask() {

    Task lowTask = new Task();
    lowTask.setPriority(TaskPriority.LOW);

    Task urgentTask = new Task();
    urgentTask.setPriority(TaskPriority.URGENT);

    int lowScore = TaskPriorityManager.calculateTaskScore(lowTask);
    int urgentScore = TaskPriorityManager.calculateTaskScore(urgentTask);

    assertTrue(urgentScore > lowScore);
}
```

Test 2: Overdue Task Bonus

```java
@Test
public void shouldAddThirtyPointsForOverdueTask() {

    Task task = new Task();

    task.setPriority(TaskPriority.MEDIUM);
    task.setDueDate(LocalDateTime.now().minusDays(1));

    int score = TaskPriorityManager.calculateTaskScore(task);

    assertTrue(score >= 50);
}
```

Test 3: Critical Tag Bonus

```java

    assertNotNull(score);
}
```

Fix

```java

    task.getUpdatedAt(),
    LocalDateTime.now()
```

Part 4: Integration Test

```java
@Test
public void shouldReturnHighestPriorityTasks() {

    Task task1 = new Task();
    task1.setPriority(TaskPriority.LOW);

    Task task2 = new Task();
    task2.setPriority(TaskPriority.URGENT);

    Task task3 = new Task();
    task3.setPriority(TaskPriority.HIGH);

    List<Task> tasks = List.of(task1, task2, task3);

    List<Task> topTasks =
        TaskPriorityManager.getTopPriorityTasks(tasks, 2);

    assertEquals(2, topTasks.size());

    int firstScore =
        TaskPriorityManager.calculateTaskScore(topTasks.get(0));

    int secondScore =
        TaskPriorityManager.calculateTaskScore(topTasks.get(1));

    assertTrue(firstScore >= secondScore);
}
```

Reflection

Testing the TaskPriorityManager helped me understand how task scores are calculated using multiple factors such as priority, due dates, status, tags, and update dates. The most important tests focused on priority weighting, overdue tasks, and task status because these have the largest impact on the final score. I also learned that edge cases such as null due dates and recently updated tasks should be tested to ensure the application behaves correctly. The integration test demonstrated how calculateTaskScore, sortTasksByImportance, and getTopPriorityTasks work together to produce a prioritized list of tasks. Using AI helped identify additional scenarios and edge cases that could easily be overlooked during manual test planning.
